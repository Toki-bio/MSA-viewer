1. Getting Started ↑

1.1 What is MSA Viewer?

MSA Viewer is a browser-based tool for viewing, editing, and analyzing multiple sequence alignments. It handles alignments of any size — from a handful of sequences to thousands — and supports DNA, RNA, protein, and coding sequence (CDS) data. It runs entirely in your browser with no installation; an optional local Node.js server adds MAFFT realignment, BLAST search, and remote file access.

1.2 Loading an Alignment

Load data in any of these ways:

  • Paste & Load — paste MSF or FASTA text into the input box and click Load
  • Drag & Drop — drop .msf, .fasta, .fa, .aln, or .sam files onto the page
  • File Picker — click Load File or press Ctrl+L
  • URL — paste a URL to a raw alignment file and click Load URL
  • SAM/BAM/CRAM — paste SAM text directly, or load BAM/CRAM via the server endpoint
  • SSH — fetch files from a remote server via SSH (requires server running with SSH config)
  • BLAST — retrieve GenBank sequences by accession and align them on the fly

1.4 Interface Overview

The interface is organized into a collapsible menu bar at the top and the alignment viewport below:

  • File — load, save, import, export
  • Edit — undo/redo, delete, copy, sequence editing tools
  • Colour — colour schemes, custom sequence highlighting
  • Selection — select, sort, group, filter sequences
  • Analysis — dot plot, repeat finder, tree builder, codon analysis
  • Display — view mode, shading, consensus, zoom
  • Help — keyboard shortcuts, manual

Press Ctrl+H to hide/show the menu bar; press Ctrl+I for keyboard shortcuts.

2. View Modes ↑

2.1 Single Mode

Default continuous alignment view. All sequences displayed in a single scrolling pane. Best for general browsing, editing, and small-to-medium alignments ( < 100 sequences × < 5000 columns). Sequence names are on the left; residues are colour-coded by conservation.

2.2 Block Mode

Splits the alignment into fixed-width blocks (default 50 columns) for readability. Sequence names repeat in each block. Ideal for publication-quality viewing. Adjust block width with the Block Size slider.

2.3 Compact Mode (IGV-style)

Renders alignments as SVG read-packing tracks, inspired by IGV. Each read is a horizontal bar; mismatches from the reference are red ticks. A coverage histogram is shown at the top. Ideal for hundreds or thousands of reads aligned to a single reference.

Compact-mode specific controls:

  • Diffs only — reads become thin hairlines; only mismatch positions are visible. Great for spotting variants at a glance
  • Pairs — dashed lines connect paired-end reads (requires SAM input with pair flags)

2.4 Highlight Diffs

Available in all modes. When checked, positions where all sequences have the same base are dimmed to 25% opacity, drawing attention to variable sites only. Turn this on alongside Block mode for a quick "where are the mutations?" overview.

3. Display Controls ↑

3.1 Conservation Shading

Four shading modes control how residues are coloured based on conservation:

ModeDescription
IdentityColours only columns where all residues match exactly
SimilarityGroups amino acids by physico-chemical properties (BLOSUM62-based)
ClustalClustalW-style colour palette for amino acids
ZappoColours by amino acid property class (hydrophobic, positive, negative, etc.)

Threshold sliders control how dark the highlight appears: Black (highly conserved), Dark (moderately conserved), Light (weakly conserved). Adjust thresholds from the Display menu.

3.2 Colour Schemes

Built-in nucleotide colour schemes: Default, Taylor, Nucleobase. Custom colour settings via the Colour menu: manually colour individual sequences, groups, or taxonomic clades.

3.3 Consensus Line

Toggle with the Show consensus checkbox. The consensus sequence is displayed at the top or bottom of the alignment (selectable via Top / Bottom radio buttons). The consensus threshold and minimum coverage can be adjusted with sliders.

3.4 Zoom & Name Length

  • Zoom — changes font size (50%–500%)
  • Name Length — truncates sequence names to N characters. Set to 0 for full names
  • Sticky Names — lock the name column in place while scrolling horizontally

3.5 Snapshot System

Save and restore complete viewer states. A snapshot captures the current alignment, colour scheme, view mode, zoom level, selected rows/columns, search highlights, and custom colour assignments. Snapshots are stored as JSON files.

  • Save Preset — Ctrl+S — save current state
  • Load Preset — Ctrl+O — restore saved state
  • Snapshots sidebar — manage multiple snapshots; click to preview, double-click to load

4.3 GeneDoc-style Edit Mode

Toggle Edit Mode from the Edit menu to modify individual residues directly in the alignment. Three edit tools are available:

  • Type (Type) — click a cell, type a residue letter or - for gap
  • Insert Column — insert a gap column at the cursor position
  • Select Column — click or drag to select columns for bulk delete
Note: All editing operations are undoable. Edit mode shows the alignment length in the status bar so you can verify changes are applied correctly.

4.4 Sequence Text Editor (SeqEdit)

Open the full-sequence text editor from the Edit menu. This provides a textarea with the raw sequence for the selected row, plus quick transformations: Degap, Reverse, Complement, Rev Comp, Uppercase, Lowercase. Check Pad/trim to alignment length to auto-adjust length on apply.

4.5 Add Sequences

Paste FASTA sequences into the "Add Sequences" modal. Options:

  • Just Add — append sequences padded with gaps at the end
  • Add & Align — realign the full alignment with MAFFT (add-keep-length mode)
  • Align to consensus — align new sequences against the existing consensus before adding
  • Place at top — insert new sequences at position 1 (otherwise appended at bottom)

5. Analysis Features ↑

5.1 Codon Analysis (MACSE-inspired)

Activates codon-aware viewing for coding sequences (CDS). Your alignment must be nucleotide and have length divisible by 3. Enable with the Codons checkbox in the Display menu.

Select the appropriate genetic code from the dropdown (17 variants available):

  • Standard (NCBI table 1) — default for nuclear genes
  • Vertebrate Mito (table 2) — AGA/AGG = Stop, ATA = Met, TGA = Trp
  • Yeast Mito (table 3) — ATA = Met, CTN = Thr, TGA = Trp
  • Invert Mito (table 5) — AGA/AGG = Ser, ATA = Met, TGA = Trp
  • …plus 12 other mitochondrial and alternative nuclear codes

When active, features appear:

FeatureVisual
Codon positionBlue (1st), green (2nd), orange (3rd) underlines
In-frame stop codonsRed background, white bold text
FrameshiftsWavy red underline at indel positions that break the reading frame
Synonymous mutationsGreen underline — amino acid unchanged relative to reference
Non-synonymous mutationsDouble red underline — amino acid changed
AA translationRow below each sequence showing amino acid letters; stop codons in bold red (*)

5.2 Dot Plot

Self-comparison or pairwise comparison of the alignment. Opens from the Analysis menu.

  • Window size — sliding window length (odd numbers, 1–61)
  • Threshold — minimum identity for a dot to be plotted (0–100%)
  • Context radius — nucleotide context shown in the info panel when hovering (5–100 bp)
  • RevComp B — reverse-complement axis B to find inverted repeats

Additional dot plot features:

  • Region detection — automatically finds the top 30 strongest diagonal runs
  • Click-to-navigate — click any region in the sidebar to jump to that position in the alignment
  • Copy Region — export the hovered/selected region as FASTA
  • Export PNG / SVG — save the dot plot as an image for publication
  • Hover context — shows aligned sequence context with mismatch highlighting

5.3 Repeat & TSD Finder

Scans the alignment for tandem repeats and target site duplications (TSDs), useful for transposable element annotation.

  • Tandem repeats — detects repeats of configurable minimum length and copy number
  • TSD detection — finds flanking direct repeats with adjustable mismatch tolerance and window size
  • TSR Mark — highlights found TSD pairs in the alignment using colour, bold, or lowercase styles

5.4 Alignment Tree

Build a UPGMA guide tree from pairwise distances. Opens from the Analysis menu.

  • Uses pairwise identity distance (1 − fraction of identical non-gap positions)
  • Outputs Newick format with branch lengths
  • Copy Newick — copy tree string to clipboard
  • Download .nwk — save as Newick file
  • Text tree visualisation in the output panel

5.5 Search

The search bar (top-right of the menu) highlights occurrences of a sequence motif. Matches are highlighted in the alignment; total count and positions are reported. Click the RC button to search the reverse complement instead. Check the .* checkbox to interpret the motif as a JavaScript regular expression (e.g., [AG]CGT, ATG.{3}TAA). Ctrl+Click on any residue to search for that base.

5.6 Sequence Clustering (Subfamily Detection)

Clusters sequences into putative subfamilies using a position-pattern-based algorithm. Open from the Analysis → Cluster menu (or Colour → Colour by Cluster). Designed for transposable element family annotation.

How it works:

  • At each alignment column, sequences are grouped by shared nucleotide
  • Groups meeting size and quality thresholds are collected as candidates
  • Near-identical groups are fuzzy-merged (Jaccard ≥ 90%)
  • Diagnostic features are scored — perfect-unique positions get highest weight
  • Outliers matching <50% of cluster features are pruned
  • The algorithm iterates with progressively relaxed thresholds to recover smaller families

Configurable parameters:

ParameterDefaultMeaning
Min cluster size3Smallest acceptable cluster
Quality threshold (small)85%For clusters ≤10 seqs
Quality threshold (medium)75%For clusters 11–20 seqs
Quality threshold (large)65%For clusters >20 seqs
Min perfect features4Minimum diagnostic positions required
Max iterations20Progressive relaxation steps

After clustering:

  • Results show cluster sizes, diagnostic mutation tables, and perfect vs. imperfect features
  • Colour by Cluster assigns persistent background colours to sequence names by cluster membership
  • Hovering a cluster row highlights all member sequences in the alignment
  • Cluster Presets save/restore parameter configurations for reproducible analysis
  • Use Group Consensus to compute and insert a consensus row for any selected cluster

Tip: Trim alignment ends first (Analysis → Cluster → Trim Parameters) to avoid ragged-terminal artifacts inflating cluster scores with gap-based patterns.

7. Server Features ↑

The optional Node.js server (server.js) runs locally on port 3000 and provides backend services. Start it with: 

node server.js

7.1 MAFFT Integration

Realign alignments or add sequences using MAFFT. Configure the MAFFT binary path in the server settings. Supports multiple MAFFT strategies (auto, L-INS-i, G-INS-i, E-INS-i). Input is sanitized — only whitelisted MAFFT options are passed to the binary.

7.2 BLAST Search

Run BLAST directly from the viewer. Right-click a sequence name and select BLAST or use the Actions menu. Results are displayed inline with alignment previews and one-click retrieval.

  • NCBI BLAST — requires BLAST_API_KEY env var
  • Local BLAST — requires blastn / blastp in PATH and indexed databases

7.2.1 BLAST Database Management

The BLAST dialog now fetches the database list dynamically from the server. Click + Manage Databases to open a management modal where you can:

  • List databases — view all configured databases with availability status
  • Add database — upload a FASTA file, set a name and description; the server runs makeblastdb to index it automatically
  • Delete database — remove a database and its BLAST index files (.nhr, .nin, .nsq, etc.)

Database metadata is stored in blast_dbs.json in the server directory. FASTA files and indexes reside in the blast_dbs/ subdirectory. Unavailable databases (missing files or unformatted) are shown greyed out with checkboxes disabled.

7.3 SSH Remote Loading

Load alignment files directly from a remote server via SSH. Configure host, user, and key path in the server. Files are fetched with cat and piped to the browser — no temporary files written.

  • Path traversal protection — .. and special characters are blocked
  • SSH key path configurable via SSH_KEY_PATH environment variable
  • Status console shows transfer progress

7.4 BAM/CRAM Support

Endpoint: POST /api/bam2sam

Converts BAM/CRAM files to SAM text on the server using samtools view -h. The resulting SAM is sent to the browser, parsed client-side, and rendered as a read alignment in the viewer. 

# Example request body
{
  "bamPath": "/data/alignments/sample.bam",
  "region": "chr1:1000-5000"   // optional
}

7.5 Security

  • CORS — restricted to localhost by default; configurable via CORS_ORIGIN env var
  • Request body limit — 10 MB maximum
  • Path traversal blocked — .. and shell metacharacters rejected in file paths
  • MAFFT input sanitized — only whitelisted options pass to the binary
  • BLAST parameters validated — no arbitrary flag injection

9. Tips & Tricks ↑

9.1 Large Alignments

  • Use Compact mode for >200 sequences — it's designed for thousands of reads
  • Turn on Highlight Diffs to visually compress conserved regions
  • Use Block mode with smaller block sizes (20-30 columns) for wide alignments
  • For extremely large files (>500 seq × 5000 col), consider splitting the input

9.2 Publication Figures

  • Block mode + Zappo shading for amino acid alignments
  • Export SVG for vector graphics — edit in Inkscape/Illustrator
  • Export RTF for Word-compatible alignments with conservation shading
  • Dot plot → Export SVG for high-quality dot plots
  • Use Snapshots to save exact figure settings and reproduce them later

9.3 Coding Sequence Analysis

  • Ensure your alignment is in-frame (length multiples of 3) before enabling Codon mode
  • Select the correct genetic code for your organism — mitochondrial genes use different codes
  • Check for frameshifts (wavy red underlines) — these often indicate sequencing errors or pseudogenes
  • Non-synonymous mutations (double red underline) that cluster may indicate positive selection
  • Stop codons in the middle of a CDS suggest pseudogenization or misannotation
  • Use Highlight Diffs + Codon Analysis together to rapidly scan for impactful mutations

9.4 SAM/BAM Workflows

  • Paste SAM text directly into the input — auto-detected
  • For BAM files on the server, use the /api/bam2sam endpoint
  • Use Compact mode for read alignments — it shows coverage, mismatches, and paired-end connections
  • Turn on Pairs to see insert-size relationships between paired reads
  • Use Diffs only to see only variant positions across hundreds of reads

9.5 General Productivity

  • Ctrl+Click any residue to instantly search for that base throughout the alignment
  • Use Ctrl+Shift+R to realign a problematic region without touching the rest
  • Colour groups of sequences by taxonomy for visual comparison (Colour menu)
  • The Undo stack preserves all edits — freely experiment with editing
  • Keep the server running if you use MAFFT/BLAST/SSH frequently — it's lightweight

9.6 Clustering Workflow

  • Start with trimming (Analysis → Cluster → Trim Parameters) to exclude ragged alignment ends — gap patterns inflate cluster scores
  • Run clustering at default settings first, then adjust quality thresholds if clusters are too large or too fragmented
  • Use Colour by Cluster to visually validate cluster assignments — outliers are obvious against colour-uniform groups
  • For each cluster: run Group Consensus to extract the subfamily consensus sequence
  • Export consensus + member sequences to the SINE Pixel Viewer for dense copy-level annotation
  • Save parameter presets so different TE families can be re-analysed with consistent settings
